# ONNX Models

This directory contains ONNX graphs used as ATC inputs.

The base ONNX files are exported from the MediaPipe legacy/task TFLite models. The optimized palm detector ONNX graphs are generated by deterministic graph rewrites so ATC can compile numerically stable OM models for Ascend 310B.

## Optimized Palm ONNX Graphs

Production full palm OM is built from:

```text
mediapipe_legacy_0_10_14_palm_detection_full_downsample_resize_maxpool_slices.onnx
```

The 8T lite palm candidate is built from:

```text
mediapipe_legacy_0_10_14_palm_detection_lite_downsample_resize_maxpool_slices.onnx
```

This lite optimized ONNX is equivalent to the original lite ONNX at raw-output level:

| output | max_abs | mean_abs | p95_abs |
| ---: | ---: | ---: | ---: |
| `0` | `5.340576e-05` | `3.211731e-06` | `1.001358e-05` |
| `1` | `8.583069e-06` | `1.155112e-06` | `2.861023e-06` |

Only the final `*_downsample_resize_maxpool_slices.onnx` lite graph is kept for deployment reproduction. Split/debug ONNX candidates were diagnostic only and are documented in `doc/13_lite_palm_split_atc_followup.md`.

Detector preprocessing is not part of the ONNX graph. The board runtime must still reproduce MediaPipe ImageToTensor preprocessing, anchors, decode, NMS, ROI crop, and landmark projection.
